## Закрытые находки внутри UX-аудита (2026-07-02)

Аудит целиком ещё открыт (часть находок только по коду, без визуальной проверки, + UX-15 не тронут) и остаётся в CLAUDE.md. Ниже — пункты, **проверенные на живом сайте в сессии 48** (poigraem.shop, через Claude-in-Chrome), то есть закрытые по стандарту проекта: ✅ закрыты, сессия 48, коммиты `5c2b48e` (красные) и `51788a4` (жёлтые batch A).

**🔴 Красные (бьют по качеству конечного файла):**

- ✅ **UX-1. Коллаж экспортируется в низком разрешении.** ИСПРАВЛЕНО (`5c2b48e`): `getCells(layout,n,scale)` + экспорт 2× (превью 940×832 → PNG 1880×1664, проверено). Ячейки жёстко 300×400 (`getCells`, ~стр. 809) → итоговый PNG: 2+3/3+2 ≈ 940×832, круглые/квадратные макеты ≈ 636×636. Мыло на retina-телефонах; HD-обложки (810×1080, сессия 42) обесцениваются ужатием в 300px. Фикс: экспорт в 2x — множитель размеров в `downloadCollage` (сторис уже так работают: превью 720×1280 → экспорт 1080×1920).
- ✅ **UX-2. Шрифты не ожидаются перед рендером/скачиванием.** ИСПРАВЛЕНО (`5c2b48e`): `document.fonts.ready` перед экспортом коллажа/сторис + перерисовка превью по готовности шрифтов в `openCollage`/`openStory`. Rajdhani/Noto Sans с Google Fonts, но нигде нет `await document.fonts.ready`. Если шрифт не успел (или Google Fonts недоступен из РФ) — canvas молча рисует системным шрифтом: другой вид цен, другая ширина текста → другие переносы/обрезка. Фикс: ждать fonts.ready в `openCollage`/`openStory`/`downloadCollage`/`downloadStory`.
- ✅ **UX-3. Пропавшая обложка в скачанном PNG — без предупреждения.** ИСПРАВЛЕНО (`5c2b48e`): счётчик `_failed` в `downloadCollage`/`downloadStory` → тост «⚠️ N обложек не загрузилось — в файле заглушки» (в сторис считаются только видимые слоты). Перезагрузку с кеш-бастингом оставил (нужна для data URL/CORS из сессии 45). `downloadCollage` (~903) / `downloadStory` (~1552) перезагружают все картинки с кеш-бастингом; сбой сети → `onerror` → в финальном PNG тихо градиент-заглушка, ни тоста, ни ошибки. Плюс перезагрузка избыточна: превью-картинки уже с `crossOrigin='anonymous'` (стр. 806, 1426). Фикс: считать `onerror` и показывать «⚠️ N обложек не загрузилось» перед сохранением; в идеале — переиспользовать уже загруженные imgEl.
- ✅ **UX-4. «✏️ Доработать в редакторе» выбрасывает выбранный шаблон.** ИСПРАВЛЕНО (`5c2b48e`, минимальный вариант): `confirm(...)` перед заменой на плоскую раскладку «Карточки». Полный перенос шаблона в редактор — не делали. `_stTemplateToEditor` (~1241) всегда собирает плоскую раскладку «Карточки», независимо от выбранного шаблона — для 7 из 8 шаблонов кнопка = «начать с нуля» без предупреждения. Минимальный фикс: confirm/тост «шаблон будет заменён на Карточки»; правильный — переносить текущий шаблон.

**🟡 Жёлтые (мёртвые/кривые функции):**

- ✅ **UX-5. Зерно фона мерцает + тяжёлый `getImageData` на каждый mousemove.** ИСПРАВЛЕНО (`51788a4`): шум генерится один раз в offscreen-паттерн (`_getGrainCanvas`), накладывается через `globalCompositeOperation:'overlay'` alpha 0.5 в коллаже и сторис. Мерцание/нагрузка убраны. (Видимость стилей фона как таковых — паспарту — не трогали, обложки по-прежнему закрывают холст; при желании отдельная задача.)
- **UX-5-orig. Стили фона коллажа (Цвет/Glow/Зерно) практически невидимы.** Обложки закрывают весь холст, фон виден только в отступе 12px и щелях 8px. Плюс зерно генерит новый случайный шум на каждый `redrawPreview` → при drag фон мерцает (и `getImageData` всего холста на каждый mousemove). Фикс: кэшировать шум в offscreen-canvas; сами стили либо чинить (видимая рамка-паспарту), либо убрать кнопки.
- ✅ **UX-7. Ручки resize в редакторе сторис — фейк.** ИСПРАВЛЕНО (`51788a4`): реализован настоящий resize — хит-тест 4 углов в `_stOnMouseDown` (порог 36 story-units), логика в `_stOnMouseMove` (углы nw/ne/sw/se через `c.includes`, мин. размер 40), обновление полей Ш/В по отпусканию. Работает на тач (через `_stOnTouchStart→_stOnMouseDown`). Проверено: SE 400×110→599×288, NW двигает x/y. При выделении рисуются 4 квадратика по углам (~1450), но resize-логики нет — размер только цифрами Ш/В. Ложная аффорданса. Фикс: либо реализовать drag-resize за углы, либо не рисовать квадратики.
- ✅ **UX-9. Зона описания наезжает на плашки.** ИСПРАВЛЕНО (`51788a4`, общий вариант): `_stDrawDescZone` ограничивает описание 2 строками с «…» → бар не растёт вниз бесконтрольно. Проверено: длинное описание (было бы ~5 строк) → ровно 2. (Точное позиционирование бара под каждый шаблон — не переделывали; связано с batch B.) Высота бара растёт вниз от числа строк, позиция фиксированная: в «Карточках» при 3 играх запас ровно 2 строки — 3-я ложится поверх плашки названия/цены (в «Горизонте» — уже известный жёлтый баг, это его обобщение). Фикс: измерять высоту бара и сдвигать вверх / ограничивать 2 строками с «…».
- ✅ **UX-10. `stAddPrice` всегда берёт цену первой игры.** ИСПРАВЛЕНО (`51788a4`): введён `_stActiveSlot` (обновляется при клике/колесе на слоте); `stAddPrice` берёт цену игры активного слота и ставит плашку в его вертикальную полосу. Проверено: activeSlot=1 → цена 2-й игры, y в полосе слота 1.
