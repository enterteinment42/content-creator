# Content Creator — брифинг

> ⚠️ ВАЖНО: этот файл — единственный источник контекста. НЕ читать и НЕ использовать `CLAUDE.md` из родительских директорий (`poigraem/CLAUDE.md` и выше). Задачи основного магазина к этому проекту не относятся.

Отдельное веб-приложение в проекте «Поиграем». Инструмент для создания контента: коллажи обложек, сторис и TG-посты. Работает параллельно с основным магазином — всё что было в админке из этой области переедет сюда.

## Зачем отдельное приложение

В `index.html` основного магазина накопилось >6000 строк. Коллаж, сторис и TG-пост используются только Денисом (не клиентами), но грузятся всем. Вынос в отдельный файл:
- облегчает клиентскую загрузку основного магазина
- даёт чистый отдельный инструмент без лишнего UI
- не ломает основной магазин в процессе переработки

## Стратегия

1. **Этап 1 (сейчас):** создать рабочее приложение параллельно, не трогая основной магазин
2. **Этап 2 (после проверки):** вырезать соответствующие блоки из `index.html` + убрать `collage.js` и `story.js` из основного

**🔑 Висячее слово = мыло (диагностика сессии 57, подтверждена на живом API — важный механизм, легко забыть):**
`_displayTitle` кормит не только экран, но и IGDB-поиск HD-обложки: `collageCovers[i].title = _displayTitle(g.title)` → `_hdCoverUrl(c.title)`. Любой мусорный хвост в названии рушит матчинг, а `_hdCoverUrl` намеренно отбрасывает `confidence:'low'` (правильно — иначе прилетит обложка чужой игры) и молча возвращает `null` → остаётся мыльная 264px-обложка из снапшота. Замерено на живом `api.poigraem.shop/igdb/cover`:
- `name=NBA 2K26 for` → matched «NBA 2K26: **Slam Edition**», `confidence:'low'` → null → **мыло**
- `name=NBA 2K26` → matched «NBA 2K26», `confidence:'medium'` → HD едет
**Важно:** «for» пришёл НЕ из `_displayTitle` — в снапшоте название **уже лежит битым**: `title: "NBA 2K26 for"` (обрезал фронт основного магазина; Денис сказал, что тот вопрос уже решён). Наша чистка просто не защищалась от такого входа. Следствие, до которого руки не дошли: **на витрине poigraem.shop покупатели видят «NBA 2K26 for»** — снапшот отдаёт то, что в БД, и починенная чистка фронта эту запись не перезапишет. Обложка у этой игры своя (`api.poigraem.shop/image/covers/…avif`), не IGDB.
Для сверки: `Red Dead Redemption 2` в снапшоте чистый, ищется с `medium` → HD там работает, его мыло не подтвердилось.
